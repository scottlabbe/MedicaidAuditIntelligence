You’re close. Two core issues I see from your example: (1) states-with-data look “blank” because their fill matches the page, and (2) the tooltip tracks the cursor, so you can’t click links. Here’s a surgical fix list + minimal code patches.  

⸻

1) Make states visibly distinct (and still on-brand)

Problem: fill for “has data” == your card/background white → looks invisible.

Fix: add explicit map vars and use a subtle brand tint for has data, a muted gray for no data, and a clear hover color.

/* index.css (one-time) */
:root{
  --map-no-data: hsl(var(--muted) / 0.35);
  --map-has-data: hsl(var(--primary) / 0.15); /* brand tint */
  --map-hover:    hsl(var(--primary) / 0.70);
  --map-stroke:   hsl(var(--border));
}

Then in USMap.tsx geography styles:

style={{
  default: { fill: hasData ? "var(--map-has-data)" : "var(--map-no-data)", stroke: "var(--map-stroke)", strokeWidth: 0.5 },
  hover:   { fill: "var(--map-hover)" },
  pressed: { fill: "var(--map-hover)" },
}}

Bonus: add an outline layer so borders pop:

{/* After <Geographies geography={TOPO_URL}>… add another pass for borders */}
<Geographies geography={TOPO_URL}>
  {({ geographies }) => geographies.map(geo => (
    <Geography key={geo.rsmKey} geography={geo} fill="none"
      stroke="var(--map-stroke)" strokeWidth={0.6} pointerEvents="none" />
  ))}
</Geographies>


⸻

2) Let users click tooltip links (don’t chase the mouse)

Problem: tooltip follows pointer; it’s pointer-events: none, so links are unclickable.

Fix (simple & reliable): “pin” tooltip position on enter, stop updating on move, and make the tooltip interactive. Hide only when the mouse leaves both the state and the tooltip (with a tiny delay to prevent flicker).

// key changes in USMap.tsx
const [activeKey, setActiveKey] = useState<string|null>(null);
const [anchor, setAnchor] = useState<{x:number;y:number}|null>(null);
const [overTip, setOverTip] = useState(false);
const boxRef = useRef<HTMLDivElement>(null);

// On state enter: setKey + pin position once
<Geography
  onMouseEnter={(e) => {
    const r = boxRef.current!.getBoundingClientRect();
    setActiveKey(key || null);
    setAnchor({ x: e.clientX - r.left, y: e.clientY - r.top }); // PIN
  }}
  onMouseLeave={() => {
    // hide only if not hovering tooltip (defer a tick)
    setTimeout(() => { if (!overTip) { setActiveKey(null); setAnchor(null); } }, 50);
  }}
  // remove onMouseMove handler that was updating position
/>

{/* Tooltip */}
{activeKey && anchor && (
  <div
    className="absolute z-10 w-80 max-w-[calc(100%-2rem)] rounded-xl border border-border bg-popover text-popover-foreground shadow-xl p-3 pointer-events-auto"
    style={{ left: Math.min(anchor.x + 16, (boxRef.current?.clientWidth ?? 0) - 340), top: Math.max(8, anchor.y - 8) }}
    onMouseEnter={() => setOverTip(true)}
    onMouseLeave={() => { setOverTip(false); setActiveKey(null); setAnchor(null); }}
  >
    {/* links are now clickable */}
  </div>
)}

Alternative (nice UX): add click-to-pin: clicking a state toggles a persistent tooltip; Esc or click outside closes. That’s excellent for mobile too.
