You’re on the right track. From your latest mock (lots of white, light card, light map), the two biggest UX gaps are:
	1.	counties/states with data don’t visually “read” as different from the background, and
	2.	the tooltip still behaves like a hover-preview, not a thing users can interact with.  

Here’s a tight fix list with drop-in changes.

⸻

1) Give the map its own canvas (subtle gray), then tint states

Goal: reduce the wall of white and make “has data” immediately scannable—without shouting.

Add map theme tokens (once)

/* index.css */
:root{
  --map-canvas: hsl(var(--muted) / 0.35);   /* panel behind the SVG */
  --map-no-data: hsl(var(--muted) / 0.45);  /* base fill for states w/ no data */
  --map-has-data: hsl(var(--primary) / 0.18); /* brand-tinted fill for states w/ data */
  --map-hover:    hsl(var(--primary) / 0.72);
  --map-stroke:   hsl(var(--border));
}

Style the map card

// Dashboard: wrap your map in a distinct panel
<div className="rounded-2xl border border-border p-4 md:p-6 bg-[var(--map-canvas)]">
  <USMap data={data} scope={scope} />
</div>

Update state fills

// USMap.tsx (Geography style)
style={{
  default: { fill: hasData ? "var(--map-has-data)" : "var(--map-no-data)", stroke: "var(--map-stroke)", strokeWidth: 0.6 },
  hover:   { fill: "var(--map-hover)" },
  pressed: { fill: "var(--map-hover)" },
}}

Crisp borders on top (optional but helps a ton)

{/* A second pass purely for borders */}
<Geographies geography={TOPO_URL}>
  {({ geographies }) => geographies.map(g => (
    <Geography key={g.rsmKey} geography={g} fill="none" stroke="var(--map-stroke)" strokeWidth={0.6} pointerEvents="none" />
  ))}
</Geographies>

———

3) Tone down the rest of the page so the map reads first

Your “Recent Activity” and “Coverage Overview” cards compete for attention (light backgrounds, bold numbers). Give the map center stage:
	•	Use bg-card (not plain white) for stat cards, and text-muted-foreground for labels, reserving strong text for the headline number only.
	•	Add a tiny “legend” under the map: ● shaded = has recent reports (keeps cognitive load low).  

<div className="mt-3 flex items-center gap-3 text-xs text-muted-foreground">
  <span className="inline-flex items-center gap-1"><span className="h-3 w-3 rounded-sm inline-block" style={{background:'var(--map-has-data)'}} /> Has recent reports</span>
  <span className="inline-flex items-center gap-1"><span className="h-3 w-3 rounded-sm inline-block" style={{background:'var(--map-no-data)'}} /> No recent reports</span>
</div>


⸻

4) Make tiny states and DC easy to hit
	•	Keep using Albers USA, but pad the hit area for RI/DE/DC: give a larger transparent stroke so hover/focus is easier.

style={{
  default: { fill: /* ... */, stroke: "var(--map-stroke)", strokeWidth: 0.6, vectorEffect: "non-scaling-stroke" },
}}
// then add a separate "hit" geography if needed:
<Geography geography={geo} fill="transparent" stroke="transparent" style={{ default:{ strokeWidth:10, strokeOpacity:0 } }} />

	•	Optionally render a small Marker dot for DC tied to the “FED” key.

⸻

5) Guard against UI jitter & clipping
	•	Add a show delay (100ms) to reduce accidental flicker when sweeping across states.
	•	Keep the tooltip inside the container bounds (you already clamp left; clamp top too).
	•	If your map card has overflow-hidden, your pinned tooltip is fine. If you portal it to document.body, switch to viewport coordinates.

⸻

6) A11y polish (fast wins)
	•	aria-label={${STATE} — ${count} recent reports} on each Geography.
	•	Let Enter open the pinned tooltip; Escape closes it.
	•	Focus ring: add focus:outline-none focus-visible:ring-2 focus-visible:ring-[color-from-tokens] to the tooltip’s first link for keyboard users.

⸻

7) Performance niceties
	•	Memoize a Set of keys that have data (useMemo(() => new Set(Object.keys(byKey)), [byKey])) so hasData checks don’t allocate arrays.
	•	Keep your React Simple Maps geographies outside of state updates; only style depends on hasData.
