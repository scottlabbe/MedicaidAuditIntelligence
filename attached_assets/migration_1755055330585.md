# Medicaid Audit Explorer — Product & UI Design Spec

> Goal: a sharp, trustworthy, fast app that helps program managers and auditors quickly *find, compare, and act on* insights from \~105 (and growing) Medicaid audit reports from the last five years.

---

## 1) Audience, Goals, and Non‑Goals

**Primary users**

* Program managers (need quick situational awareness, trend views, and actionable recommendations).
* Auditors/reviewers (need precise provenance, filters, comparisons, and exports for workpapers).

**Top goals**

1. Find relevant audits fast (state/agency/topic/date/keyword/semantic search).
2. Understand the essence (objective, scope, conclusion, findings, recommendations) with **clear provenance**.
3. Compare across reports (by state, agency, topic, timeframe) to spot patterns.
4. Export or copy clean summaries and citations for reuse.

**Non‑goals (for now)**

* Full document management of non-audit content.
* Freeform discussion threads (consider later).

**Project constraints (from you)**

* DB already exists (Postgres + SQLAlchemy) following a star schema centered on `reports`.
* Small private audience (\~12 users). **No user accounts** required.
* Hosted on **Replit**. Frontend: **Next.js + Tailwind + shadcn/ui + lucide**. Backend: **Python** (FastAPI).
* Need clear guidance on **CORS**, **secure API exposure**, and **DB permissions**.

## 5) Backend Stack (Python)

**FastAPI + SQLAlchemy/SQLModel + Postgres** (keep your ORM). Add `alembic` for migrations and `uvicorn` for dev.

**Security posture (no user accounts)**

* **Two classes of endpoints**:

  1. **Public read**: `/reports`, `/reports/{id}`, `/dash/*`, `/entities/*` (read‑only) → rate‑limited, CORS‑restricted.
  2. **Private ops**: `/admin/*`, `/ingest/*` → protected by **server‑only API key** (HMAC Bearer) and **not exposed to the browser**.
* Expose private ops **only** through Next.js **server routes** (so the API key never reaches the client). Frontend browser calls your Next.js route, Next calls FastAPI with the secret.

**CORS & headers**

* Allow origins: the exact Next.js domain(s) only (Replit app URL + custom domain). No `*`.
* Deny credentials; allow `GET, OPTIONS` for public routes; `POST/PUT` only for server‑to‑server.
* Add security middleware: `HTTPSRedirectMiddleware`, `TrustedHostMiddleware`, and secure headers.

**Example FastAPI security/cors setup**

```python
from fastapi import FastAPI, Depends, HTTPException, status, Request
from fastapi.middleware.cors import CORSMiddleware
from starlette.middleware.httpsredirect import HTTPSRedirectMiddleware
from starlette.middleware.trustedhost import TrustedHostMiddleware
import hmac, hashlib, os

ALLOWED_ORIGINS = os.getenv("ALLOWED_ORIGINS", "").split(",")  # e.g. https://yourapp.replit.app,https://yourdomain.com
TRUSTED_HOSTS = os.getenv("TRUSTED_HOSTS", "").split(",")    # e.g. yourapp.replit.app,yourdomain.com
API_SECRET = os.getenv("INTERNAL_API_SECRET")

app = FastAPI()
app.add_middleware(HTTPSRedirectMiddleware)
app.add_middleware(TrustedHostMiddleware, allowed_hosts=TRUSTED_HOSTS + ["*"] if not TRUSTED_HOSTS[0] else TRUSTED_HOSTS)
app.add_middleware(CORSMiddleware,
    allow_origins=ALLOWED_ORIGINS,
    allow_credentials=False,
    allow_methods=["GET", "OPTIONS"],
    allow_headers=["Authorization", "Content-Type"],
)

def require_hmac(request: Request):
    # Expect header: Authorization: Bearer <HMAC_SHA256(timestamp.path, secret)>::<timestamp>
    header = request.headers.get("authorization", "")
    try:
        scheme, token = header.split(" ", 1)
        mac, ts = token.split("::", 1)
        body = f"{ts}{request.url.path}".encode()
        if not hmac.compare_digest(mac, hmac.new(API_SECRET.encode(), body, hashlib.sha256).hexdigest()):
            raise ValueError
    except Exception:
        raise HTTPException(status_code=status.HTTP_401_UNAUTHORIZED)

@app.get("/health")
async def health(): return {"ok": True}

@app.get("/admin/reindex", dependencies=[Depends(require_hmac)])
async def reindex(): return {"status": "ok"}
```

## 6) Frontend Stack (Next.js + Tailwind + shadcn/ui)

**Next.js recommendations**

* Use **App Router**. Server Components for Explore & Detail; Client Components for filters.
* Create **Route Handlers** (`/app/api/*`) as a thin proxy to FastAPI for any private ops (inject HMAC in server only).
* Add **Cache-Control** for list endpoints: `s-maxage=300, stale-while-revalidate=86400`.
* Use `@tanstack/react-query` or `next` built‑in caching for incremental fetching.
* Components: shadcn/ui (Card, Badge, Tabs, Sheet for filters), lucide icons.

**CORS note**: Browser only calls **Next.js** origin; Next.js calls FastAPI with secret → avoids leaking secrets.

13) Security & Compliance (small private app, no user accounts)

Network & hosting
	•	Host Next.js and FastAPI as separate services on Replit (or two repls). Expose FastAPI only over HTTPS. Prefer a custom domain for each.
	•	Set CORS to your Next.js origin(s) only.
	•	All secrets via Replit Secrets; never commit .env.

API access model
	•	Public read endpoints only (GET) → rate limited and CORS‑restricted.
	•	Private ops (admin/ingest) → HMAC Bearer token required and only callable from Next.js server routes.
	•	Add simple IP allowlist at the DB level if your provider supports it; otherwise rely on strong creds + TLS + least‑privilege roles.

Headers & TLS
	•	Force HTTPS; set Strict-Transport-Security (HSTS), X-Content-Type-Options: nosniff, Referrer-Policy: no-referrer-when-downgrade, X-Frame-Options: DENY, Permissions-Policy (turn off camera/mic/geolocation).

Rate limiting
	•	Add a lightweight middleware (e.g., Starlette limiter or a Redis-backed counter) to cap requests (e.g., 60 req/min/IP) on public endpoints.

Observability
	•	Sentry for both frontend & backend; request ID in logs; structured JSON logs.

24) Risks & Mitigations

	•	Replit domain changes → set ALLOWED_ORIGINS as env and update when domain changes; consider a custom domain.
	•	Search performance → use materialized tsvector & proper indexes; paginate aggressively.
	•	Data ambiguity across programs → add program mapping table and UI chips.
